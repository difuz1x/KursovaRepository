# TaskManager

Легка React + Vite аплікація для керування домашніми завданнями.

Коротко:
- Додавайте завдання з пріоритетом та дедлайном.
- Фільтрація за статусом, сортування за датою/пріоритетом.
- Графіки пріоритетів та статусу (Chart.js).
- Підтвердження при видаленні та очищенні, Undo для видалених/очищених завдань.

Запуск локально:

# TaskManager

Легка React + Vite аплікація для керування завданнями (локально в браузері).

Цей README додає детальну інструкцію як запустити проєкт, як імпортувати приклад файлу і короткі нотатки щодо релізу v1.0.0.

## Швидкий старт

1) Склонуйте (якщо ще не склонили) та перейдіть у папку проекту:

```powershell
git clone https://github.com/difuz1x/KursovaRepository.git
Set-Location -Path 'd:\KursovaRepository\TaskManager'
```

2) Встановіть залежності:

```powershell
npm install
```

3) Запустіть dev‑сервер (HMR):

```powershell
npm run dev
```

4) Відкрийте у браузері URL, який виведе Vite (наприклад `http://localhost:5173` або `http://localhost:5174`).

5) Побудова production:

```powershell
npm run build
```

---

## Імпорт/експорт задач (швидкий спосіб протестувати)

- У папці `public/` є приклад файлу `sample-tasks.json`.
- Коли dev‑сервер запущений, файл доступний за адресою:

```
http://localhost:5174/sample-tasks.json
```

- Щоб протестувати імпорт:
  1. У UI натисніть кнопку «Імпорт» (або виберіть відповідний елемент меню).
  2. Виберіть `sample-tasks.json` зі свого робочого столу або прямого шляху (файл також доступний у `public/`).
  3. Перегляньте прев'ю, відмітьте дублікати та натисніть Merge або Replace.

## Структура даних (коротко)

- `TaskType` (основні поля):
  - `id: string`
  - `title: string`
  - `description?: string`
  - `priority: 'low' | 'medium' | 'high'`
  - `dueDate?: string` (ISO)
  - `isCompleted: boolean`
  - `createdAt?: string` (ISO)

При імпорті файлів застосунок валідовує дані через `zod` та нормалізує їх у цю форму.

## Додаткові нотатки

- Якщо у вас є старі записи в `localStorage` з полем `deadline` або рядковим `status`, вони автоматично мігруються у `dueDate` (ISO) та `isCompleted` (boolean) при завантаженні застосунку.
- Якщо інші розробники працювали з гілкою main і ви побачите проблеми після форс‑пуша, скористайтесь `git fetch` + `git reset --hard origin/main` або переконайтесь, що ви знаєте потрібні кроки для синхронізації.
